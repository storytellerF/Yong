When a key parameter changes, it is a signal that the previous invocation is now invalid. In certain cases, it may be required to pass `Unit` as a key to one of these functions, indicating that the invocation never becomes invalid. Using `Unit` as a key should be done infrequently, and should always be done explicitly by passing the `Unit` literal. This inspection checks for invocations where `Unit` is being passed as a key argument in any form other than the `Unit` literal. This is usually done by mistake, and can harm readability. If a Unit expression is being passed as a key, it is always equivalent to move the expression before the function invocation and pass the `Unit` literal instead. !!!
